## Web Sources Checked

### YouTube

- YouTube Help, "Use automatic captioning": https://support.google.com/youtube/answer/6373554
  - Automatic captions are generated by speech recognition / machine learning.
  - The current supported language list includes Chinese, Japanese, and Korean.
  - Auto captions can fail because processing is not ready, the video is too long, audio is poor, YouTube does not recognize speech, there is long silence, overlapping speech, or multiple languages.
  - Live automatic captions are English-only.
- YouTube Help, "Manage caption settings": https://support.google.com/youtube/answer/100078
  - Viewer setting "Include auto-generated captions" applies when available and when a video does not have captions added.
- YouTube Help, "Add subtitles & captions": https://help.youtube.com/support/youtube/bin/answer.py?answer=100076
  - Creators can upload caption files, type captions, or use auto-sync.
  - Automatic captions are in the video's default language only.

Conclusion from YouTube sources:

- Chinese auto captions are officially possible for uploaded long-form videos.
- They are not reliable enough for this pipeline.
- YouTube auto-caption absence is expected for many real videos and does not imply a bug in our extractor.

### Bilibili

- yt-dlp issue, "bilibili.com better subtitle handling": https://github.com/yt-dlp/yt-dlp/issues/14463
  - Gives examples of Bilibili videos with automatic English subtitles, automatic Chinese subtitles, and both automatic English and Chinese subtitles.
  - Notes Bilibili AI subtitles may be labeled like `ai-en` and `ai-zh`.
  - Notes current yt-dlp behavior does not classify Bilibili AI subtitles exactly like YouTube auto captions.
- yt-dlp issue, "No subtitles downloaded except danmaku": https://github.com/yt-dlp/yt-dlp/issues/5446
  - Shows Bilibili page/API subtitle metadata can include `subtitle.list`, `lan`, `lan_doc`, `subtitle_url`, `ai_type`, and `ai_status`.
  - This is evidence that Bilibili subtitles can be extracted from structured metadata, but yt-dlp support has had gaps.
- BibiGPT Bilibili Subtitle Downloader: https://docs.bibigpt.co/en/function-usage/platform-function/bilibili-subtitle-downloader
  - Third-party service claims to extract Bilibili subtitles and use AI transcription if official captions are missing.
  - Useful as ecosystem evidence, not a primary source for our pipeline.

Conclusion from Bilibili sources:

- Bilibili AI subtitles are real.
- `yt-dlp` can see some Bilibili subtitle structures, but handling is less mature than YouTube.
- Bilibili subtitle extraction may need login cookies and may be unstable across videos.

### Whisper / ASR

- OpenAI Whisper release: https://openai.com/research/whisper/
  - Whisper is multilingual ASR trained on 680,000 hours of multilingual and multitask supervised data.
  - It supports transcription in multiple languages and translation into English.
  - It is robust to accents, background noise, and technical language compared with narrower systems, but still not perfect.
- OpenAI Whisper GitHub: https://github.com/openai/whisper
  - Provides command-line and Python usage.
  - Requires `ffmpeg`.
  - Lists model sizes and speed/VRAM tradeoffs.
  - Notes `large` requires about 10 GB VRAM; `turbo` is faster but not suitable for translation tasks.
- OpenAI Speech-to-text API docs: https://platform.openai.com/docs/guides/speech-to-text
  - Supports transcriptions and translations.
  - Current transcription endpoint supports `whisper-1`, `gpt-4o-mini-transcribe`, `gpt-4o-transcribe`, and `gpt-4o-transcribe-diarize`.
  - Can transcribe in the source language or translate to English.
  - File uploads are limited to 25 MB, so long videos need chunking.
- FunASR / Paraformer-zh model card: https://huggingface.co/funasr/paraformer-zh
  - Chinese ASR model with timestamps, trained on 60,000 hours of Mandarin.
  - Provides VAD and punctuation restoration models.
- Fun-ASR: https://github.com/FunAudioLLM/Fun-ASR
  - Newer Chinese/East-Asian-focused ASR family.
  - Claims support for Chinese, English, Japanese, Chinese dialects, regional accents, and multi-language recognition.

Conclusion from ASR sources:

- ASR is the only fallback with near-complete coverage because it works from audio.
- For Chinese specifically, evaluate both Whisper-family and Chinese-focused ASR such as FunASR/Paraformer/SenseVoice/Fun-ASR.
- ASR output must be treated as generated text requiring quality checks, not as authoritative source captions.

Important safety note:

- I did not use browser cookies or `--cookies-from-browser`.
- Reading browser cookies and sending them to Bilibili would use sensitive login/session data and needs explicit user authorization.

Practical implication:

- Bilibili is viable only if the video has a mirror and if the workflow can use an authenticated Bilibili session.
- For unauthenticated automated processing, Bilibili is not reliable enough as the only fallback.

## Decision Reasoning

The two proposed approaches solve different failure modes:

1. Bilibili is a platform-subtitle fallback. It can preserve human or platform-generated captions when available, but it has low recall because it depends on mirror existence, login, and per-video subtitle availability.
2. Whisper/ASR is an audio fallback. It has much higher coverage because every playable video has audio, but quality varies and must be reviewed or scored.

The robust pipeline should use both:

1. YouTube manual/generated subtitles via `youtube-transcript-api`.
2. Bilibili subtitles if a mirror exists and authenticated subtitle extraction is allowed.
3. ASR from audio as the universal fallback.

## Open Implementation Questions

- Whether to allow authenticated Bilibili subtitle extraction using cookies. This should be opt-in.
- Which ASR model should be the first local/default Chinese model:
  - Whisper large-v3 / faster-whisper for broad multilingual support.
  - FunASR Paraformer/SenseVoice for Mandarin-first quality.
  - OpenAI `gpt-4o-transcribe` for hosted higher-quality transcription.
- How to score transcript quality automatically:
  - language detection confidence
  - segment density
  - repeated/hallucinated text detection
  - duration coverage
  - spot check against description/title keywords
  - CER/WER against manual subtitles on videos that have them
