# Chinese Subtitle Strategy — Recommendations

For Mandarin Chinese videos that have neither manual nor auto-generated subtitles on YouTube. Companion to `chinese-subtitle-process-log.md` (sibling) which contains the full reasoning, live tests, and sources.

## Bottom line

| Option | Verdict | Use when |
|---|---|---|
| **Bilibili AI subs** | ✅ Possible, ⚠️ requires login cookie | A Bilibili mirror exists for the same content. Use as the **first-tier fallback** when available. |
| **Whisper (local or cloud)** | ✅ Works, ⚠️ ~2× worse than Paraformer on Mandarin (19% vs 11% WER) | Need a no-auth, no-platform-dependency fallback. **Use the Groq cloud API** for cheap fast access, or local for offline/privacy. |
| **FunASR / Paraformer-large-zh** *(new option that emerged from research)* | ✅ Best Chinese accuracy, ✅ open-source, ✅ free | The default Mandarin ASR for our pipeline. Outperforms Whisper by ~2× on Chinese benchmarks per Alibaba's own report. |

**Recommended architecture: a 3-tier fallback chain.** Bilibili first when applicable, Paraformer locally as the default, Groq Whisper as the no-GPU/cloud failover.

## Option 1 — Bilibili AI subtitles

### Possibility: Yes — verified

Bilibili exposes AI-generated subtitles tagged with the prefix `ai-` (e.g. `ai-zh`, `ai-en`, `ai-ja`). The platform-side priority order is **manual CC > AI subs > Whisper fallback** for tools that integrate with it.

### Reliability: Moderate

- **Coverage is not universal.** Creator can opt out, very short clips often skip, processing has a delay for new uploads. Most longer creator-uploaded technical / educational videos do have it.
- **Authentication required.** Verified by direct test on `BV1T7zzBQEaA`: anonymous calls to Bilibili's player API return `subtitles: []` even when the AI track exists. yt-dlp also returns no subtitles without `--cookies-from-browser`.
- **Quality is generally decent** for clear single-speaker tech content. No public WER benchmark, but third-party transcript tools treat it as a real source-of-truth tier rather than a Whisper backup.

### Steps

1. **Get a `sessdata` cookie:**
   - Log in to bilibili.com in any browser.
   - Open devtools → Application → Cookies → `bilibili.com`.
   - Copy the value of `SESSDATA` (also note `bili_jct`, `buvid3` if you want write access — read access only needs `SESSDATA`).
   - Cookie expires ~30 days; needs occasional refresh.

2. **Programmatic fetch with `bilibili-api`:**
   ```python
   # pip install bilibili-api-python
   import asyncio
   from bilibili_api import video, Credential

   credential = Credential(sessdata="<your_sessdata>")
   v = video.Video(bvid="BV1T7zzBQEaA", credential=credential)
   info = await v.get_info()
   cid = info["cid"]
   subs = await v.get_subtitle(cid)
   # subs["subtitles"] is a list of {lan, lan_doc, subtitle_url, ai_status, ai_type, ...}
   # Filter for ai_type != 0 to get AI-generated tracks.
   ```

3. **Or via yt-dlp with browser cookies:**
   ```bash
   yt-dlp --skip-download \
          --write-auto-subs --write-subs \
          --sub-langs "ai-zh,zh-CN,zh-Hans,zh-Hant" \
          --cookies-from-browser chrome \
          --convert-subs srt \
          "https://www.bilibili.com/video/BV.../"
   ```
   Note: `yt-dlp` issue [#14463](https://github.com/yt-dlp/yt-dlp/issues/14463) — `ai-` subs aren't automatically promoted to `--write-auto-subs`, so passing them explicitly via `--sub-langs` is the workaround.

4. **Cross-source linking.** When the testset already has paired YouTube + Bilibili URLs (as ours does), check the Bilibili side first. The metadata pipeline can record `transcript_source: bilibili_ai` vs `transcript_source: youtube_manual` etc.

### Pros

- Free, fast, no compute required.
- AI subs are generated by Bilibili from clean source audio with creator context — usually ≥ Whisper output quality.
- Works for content that exists *only* on Bilibili (most mainland Chinese creators).

### Cons

- Cookie management (extraction, expiry, occasional refresh).
- Not all Bilibili videos have AI subs.
- Geo: licensed content (anime, dramas) is region-locked; user-uploaded tech content is usually accessible globally without a VPN. Empirically the user already has 4 Bilibili links in their testset, so access is not the blocker.
- yt-dlp's incomplete `ai-` handling means a tiny bit of glue code per workflow.

## Option 2 — Whisper

### Possibility: Yes — well established

OpenAI's Whisper supports Mandarin (language code `zh`). Multiple deployment paths:

- **Local CPU** — slow but free.
- **Local Apple Silicon (Metal acceleration)** — comfortable on M1/M2/M3/M4 with 16 GB+. ~1× real-time for `large-v3`, ~4× real-time for `large-v3-turbo` on M1; faster on later chips.
- **OpenAI API** — `whisper-1`, $0.006/min ≈ $0.36/hour, 25 MB upload limit (chunk longer audio).
- **Groq cloud API** — `whisper-large-v3-turbo`, **$0.04/hour**, **164–216× real-time**. Same model, ~10× cheaper than OpenAI, dramatically faster. Generous free tier.

### Reliability: Moderate

The honest accuracy picture, per Alibaba's [Fun-ASR technical report](https://arxiv.org/html/2509.12508v3):

| Model | Avg WER on Chinese (6 test sets) |
|---|---|
| Whisper-large-v3 | 19.19% |
| Paraformer-v2 (Alibaba) | 10.91% |

Whisper-large-v3 is functional for Mandarin but ~2× worse than the Chinese-specialized alternative. It produces readable transcripts; expect a meaningful error rate on technical jargon, homophones, named entities. Quality is much better with a domain-specific `--initial_prompt` to anchor the vocabulary.

### Steps (Groq cloud — recommended Whisper path)

1. Sign up at [console.groq.com](https://console.groq.com) — free tier covers significant personal use.
2. Extract audio with yt-dlp (which handles both YouTube and Bilibili):
   ```bash
   yt-dlp -x --audio-format mp3 \
          -o "audio_%(id)s.%(ext)s" \
          "<URL>"
   ```
3. Send to Groq Whisper:
   ```python
   from groq import Groq
   client = Groq(api_key="...")
   with open("audio_xxx.mp3", "rb") as f:
       resp = client.audio.transcriptions.create(
           file=("audio.mp3", f.read()),
           model="whisper-large-v3-turbo",
           language="zh",
           prompt="繁體中文。視頻主題：AI agent、LLM、編程、技術評論。",
       )
   print(resp.text)
   ```
4. **Important Mandarin-specific tweaks:**
   - Pass `language="zh"` explicitly (Whisper sometimes mis-detects as `yue` or `ja`).
   - Set `prompt=` to a few sentences that include the video title, the channel's typical vocabulary, and the variant (Traditional vs Simplified). This anchors homophone resolution and named entities.
   - Whisper can output Simplified or Traditional inconsistently; if Traditional output is needed, post-process with [OpenCC](https://github.com/BYVoid/OpenCC) (`s2t`).
   - For >25 MB audio: split with ffmpeg by silence or fixed chunks and concatenate transcripts.

### Steps (local Whisper)

```bash
pip install openai-whisper
whisper audio_xxx.mp3 --model large-v3-turbo --language zh --output_format srt
```

Or with `mlx-whisper` for native Apple Silicon acceleration:
```bash
pip install mlx-whisper
mlx_whisper audio_xxx.mp3 --model mlx-community/whisper-large-v3-turbo --language zh
```

### Pros

- Universal — works for any video on any platform, given the audio.
- No cookies, no auth, no platform-specific glue.
- Privacy: local mode never leaves your machine.
- Mature ecosystem, well-documented.

### Cons

- ~2× worse than Chinese-specialized models on Mandarin.
- Local Whisper requires non-trivial RAM (8 GB minimum, 16 GB comfortable) and is slow on CPU.
- Cloud Whisper has a 25 MB per-request limit (need to chunk longer audio).
- Without a domain prompt, technical jargon and named entities take a noticeable hit.

## Option 3 (recommended default) — FunASR / Paraformer-large-zh

Not in the user's original two-option list, but emerged from research as the strictly better local option for Mandarin specifically.

### What it is

[FunASR](https://github.com/modelscope/FunASR) is Alibaba Tongyi Lab's open-source ASR toolkit. The flagship Paraformer-large-zh model:
- **60,000 hours of Chinese training data** (vs Whisper's much smaller Chinese share).
- **Non-autoregressive architecture** — ~3× faster than Whisper-style models at similar accuracy.
- **10.91% WER vs Whisper-large-v3's 19.19%** on Chinese benchmarks (Alibaba's report).
- Built-in voice-activity detection, punctuation restoration, optional speaker diarization.
- 13M+ downloads on ModelScope.

### Steps

```bash
pip install funasr modelscope torch
```

```python
from funasr import AutoModel

# First call downloads model weights (~800 MB)
model = AutoModel(
    model="paraformer-zh",      # main ASR
    vad_model="fsmn-vad",       # split long audio at silences
    punc_model="ct-punc",       # add punctuation
    spk_model="cam++",          # speaker diarization (optional)
)

result = model.generate(
    input="audio_xxx.mp3",
    batch_size_s=300,           # process in 5-min chunks
    hotword="OpenAI Anthropic Claude Gemini Agent LLM transformer",  # domain anchoring
)
print(result[0]["text"])
```

The `hotword` parameter is Paraformer's equivalent of Whisper's `initial_prompt` — pass a space-separated list of in-domain proper nouns and technical terms to lock them in.

### Pros

- Best open-source accuracy for Mandarin, by a clear margin.
- Same family as Alibaba Cloud's commercial Mandarin ASR backend (so you're using a battle-tested model, not a research artifact).
- Free, local, no auth, no upload limit.
- Punctuation + diarization without extra services.

### Cons

- One extra dependency (PyTorch + ModelScope + funasr ~1 GB total install).
- Slower first run while it downloads weights.
- Less English-multilingual than Whisper — for English videos, stay with Whisper.

## Recommended pipeline

A 3-tier fallback that exploits each option's strengths:

```
For a Chinese video where YouTube has no usable transcript:

1. If the testset has a paired Bilibili URL:
     → fetch via bilibili-api with sessdata cookie
     → check for "ai-zh" track (or "zh-CN" manual)
     → if found: transcript_source = bilibili_ai (or bilibili_manual). DONE.

2. Else (no Bilibili mirror, or Bilibili has no subs either):
     → yt-dlp -x to extract audio
     → run Paraformer-large-zh locally with hotword=video title + channel keywords
     → transcript_source = paraformer_local. DONE.

3. If Paraformer is unavailable (no GPU, model not yet installed, etc.):
     → upload audio to Groq Whisper-large-v3-turbo with language="zh" + domain prompt
     → transcript_source = whisper_groq. DONE.

For non-Chinese videos with no transcript:
     → skip step 1 (Bilibili is China-specific)
     → step 2: Whisper-large-v3 is fine (19% WER on Chinese, but much better on most other languages)
     → step 3: Groq fallback as above
```

## Migration vs greenfield

If we're adding this to the existing pipeline incrementally, the order to roll out is:

1. **Add Groq Whisper as the universal fallback first.** It's the smallest dep (just an API key) and immediately solves the "8 testset videos with no transcript" problem.
2. **Add FunASR/Paraformer when you're ready to take a local-model dep.** It improves Chinese accuracy from 19% to 11% WER for free.
3. **Add Bilibili AI subs last.** It's the most operational complexity (cookie management) for the smallest universal benefit (only helps when a mirror exists), but it's the highest-quality source when it does apply.

If we're starting from scratch and only have time for one tier, **pick Groq Whisper** — universally applicable, ~free, takes a few hours to wire up.

## Decisions to make / open questions for the user

- Are you willing to maintain a Bilibili `sessdata` cookie (expires ~30 days, needs refresh)? If yes, tier 1 is worth wiring up; if no, skip it and rely on tiers 2-3.
- Do you have 16 GB RAM on your Mac? (You probably do — needed for comfortable Paraformer-large-zh local execution.)
- Privacy preference: are you OK uploading audio to Groq, or does everything need to stay local?

These choices change which tiers to enable but don't change the architecture.
